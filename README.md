# ИИ-тьютор для сельских школьников (MVP)

## Идея продукта
Сельские школы часто испытывают нехватку учителей-предметников.
Это приложение даёт ученику 8–11 класса возможность:
1. Выбрать язык интерфейса (русский/казахский) — от этого зависит и
   язык вопросов, и структура предметов (см. ниже), и зарегистрироваться,
   чтобы сохранять свой прогресс между занятиями.
2. Пройти диагностический тест сразу по всем 13 предметам школьной
   программы Казахстана и увидеть карту сильных/слабых сторон.
3. Пройти короткий пробный тест по конкретному предмету и классу.
4. Получить не просто оценку, а **персональное объяснение** каждой
   ошибки от ИИ-тьютора (на том же языке, что и вопрос) и **план
   обучения с реальными ресурсами** для повторения (Mektep-online,
   BilimLand, Umschool).
5. Следить за прогрессом на карте по темам и смотреть накопленный
   "банк ошибок" по всем пройденным тестам.
6. Видеть на главном экране персональный дашборд: рекомендацию на
   сегодня, серию дней подряд с занятиями (🔥 streak) и общий прогресс —
   а для новых учеников, у которых ещё нет данных, вместо дашборда
   показывается логотип и мотивирующая цитата.

## Фирменный стиль BilimK
Интерфейс оформлен в фирменной палитре бренда:
- Бирюзовый `#27A295` — кнопки, полосы прогресса, успешные темы, ссылки.
- Тёмно-лазурный `#11547D` — заголовки, меню, логотип.
- Белый — фон и карточки.

Цвета заданы через `.streamlit/config.toml` (тема Streamlit) и
дополнительные CSS-классы в `branding.py`. Ключевой визуальный мотив —
**гора** (эмблема BilimK): она используется в шапке приложения и в виде
буквальной визуализации общего прогресса на странице "Мой прогресс"
(точки поднимаются по склону горы по мере роста освоенности материала,
на вершине появляется флаг при 100%).

## Двуязычный интерфейс и структура предметов
Язык интерфейса определяет ТИП ШКОЛЫ (структуру предметов языка/
литературы), а не только перевод текста:

| | Русский интерфейс (русская школа) | Қазақ интерфейсі (қазақ мектебі) |
|---|---|---|
| Русский язык/литература | отдельно | вместе ("Орыс тілі мен әдебиеті") |
| Казахский язык/литература | вместе ("Қазақ тілі мен әдебиеті") | отдельно |

"Предметные" дисциплины (Алгебра, Геометрия, Физика, Химия, Биология,
География, История Казахстана, Всемирная история, Информатика)
показываются на выбранном языке интерфейса. Английский язык — всегда
на английском, независимо от интерфейса.

**Честный компромисс по объёму:** вопросы по "предметным" дисциплинам
на русском — по 5 на класс (как и раньше), на казахском — по 3 на
класс. Перевод ~180 вопросов на казахский вручную за один заход не
позволил бы гарантировать точность терминологии по всем предметам
(особенно в химии/физике), поэтому банк можно наращивать по частям.

## Предметы в банке вопросов (8-11 класс)
Алгебра, Геометрия, Физика, Химия, Биология, География, История
Казахстана / Қазақстан тарихы, Всемирная история / Дүниежүзі тарихы,
Информатика, Русский язык, Русская литература, Қазақ тілі, Қазақ
әдебиеті, Орыс тілі мен әдебиеті, Ағылшын тілі / Английский язык —
всего 280 вопросов в русской структуре, 188 в казахской.

## Запуск (локально)
```bash
pip install -r requirements.txt
cp .env.example .env      # скопируйте шаблон
# откройте .env и вставьте свой ключ OpenRouter вместо your_api_key_here
streamlit run app.py
```
Ключ никогда не хранится в коде — только в `.env` (он в `.gitignore` и не попадает в Git).

## Деплой на Streamlit Cloud
`.env` на Cloud не работает — вместо него нужно задать секрет в интерфейсе:
`Settings → Secrets` и добавить строку:
```
OPENROUTER_API_KEY = "ваш_ключ"
```
Приложение само подхватит его через `st.secrets` (см. `ai_tutor.py`).

## Структура таблиц SQLite

### Таблица `users` — аккаунты учеников
| Поле                | Тип     | Описание                                          |
|---------------------|---------|-----------------------------------------------------|
| id                  | INTEGER | Первичный ключ                                     |
| username            | TEXT    | Уникальный логин                                   |
| password_hash       | TEXT    | Хеш пароля (pbkdf2_hmac, НЕ открытый текст)         |
| salt                | TEXT    | Индивидуальная соль для хеширования                |
| created_at          | TEXT    | Дата регистрации                                   |
| last_diagnostic_at  | TEXT    | Дата последней пройденной диагностики (NULL, если не проходил) |
| streak_days         | INTEGER | Текущая серия дней подряд с занятиями (для дашборда) |
| last_practice_date  | TEXT    | Дата последнего пройденного теста (для расчёта серии) |

### Таблица `tests` — одна строка = одно прохождение теста
| Поле             | Тип     | Описание                              |
|------------------|---------|----------------------------------------|
| id               | INTEGER | Первичный ключ                         |
| student_name     | TEXT    | Имя ученика                            |
| username         | TEXT    | Логин аккаунта (для карты прогресса)   |
| subject          | TEXT    | Предмет                                |
| grade            | INTEGER | Класс (8-11)                           |
| is_diagnostic    | INTEGER | 1, если это часть диагностического теста |
| total_questions  | INTEGER | Всего вопросов в тесте                 |
| correct_answers  | INTEGER | Количество правильных ответов          |
| created_at       | TEXT    | Дата и время прохождения (ISO-формат)  |

### Таблица `answers` — одна строка = один ответ в рамках теста
| Поле            | Тип     | Описание                                          |
|-----------------|---------|-----------------------------------------------------|
| id              | INTEGER | Первичный ключ                                     |
| test_id         | INTEGER | Внешний ключ на `tests.id`                         |
| question_text   | TEXT    | Текст вопроса                                      |
| topic           | TEXT    | Тема вопроса (для карты прогресса и аналитики)     |
| student_answer  | TEXT    | Ответ, который выбрал ученик                       |
| correct_answer  | TEXT    | Правильный ответ                                   |
| is_correct      | INTEGER | 0 / 1 — правильно ли ответил ученик                |
| ai_explanation  | TEXT    | Объяснение от ИИ-тьютора (NULL, если ответ верный)  |

Такая структура позволяет строить карту прогресса (`get_progress_map`)
по предмету И теме одновременно, а не только "средний балл по тесту".

## Логика ИИ-тьютора (кратко, для защиты)
- ИИ вызывается **только для неправильных ответов** — экономия токенов и времени.
- В промпт передаётся уже известный правильный ответ — модель объясняет
  расхождение, а не "придумывает" решение заново (снижает риск галлюцинаций).
- Промпт задаёт короткий, дружелюбный тон, подходящий для школьников.
- Есть fallback-ответ на случай сбоя API, чтобы демо не сломалось на сцене.

## Возможные направления развития (roadmap)
- Расширение банка вопросов на казахском языке до паритета с русским
  (сейчас 3 вопроса на класс вместо 5 по "предметным" дисциплинам —
  см. раздел про двуязычность выше).
- Расширение банка вопросов по всем предметам (интеграция с реальной
  программой Минобра), в том числе для "Русской литературы" и
  "Русского языка", где сейчас часть вопросов дублируется между банками.
- Динамическая генерация вопросов через LLM с учётом уровня ученика.
- График прогресса по времени (сейчас карта прогресса показывает
  срез "сейчас", а не динамику по датам).
- Оффлайн-режим / работа при слабом интернете (актуально для сельской местности).
- Сброс пароля / восстановление доступа к аккаунту (сейчас не реализовано).
